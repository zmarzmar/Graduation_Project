# AGENTS.md

## Scope
- 이 저장소는 `FE/` 프론트엔드와 `BE/` 백엔드로 구성된다.
- 공통 규칙은 이 문서에 둔다.
- 반복 절차와 검증 체크리스트는 skill로 분리한다.

## Architecture Boundaries
- 프론트엔드의 백엔드 호출은 `FE/lib/api.ts`를 단일 진입점으로 유지한다.
- 프론트엔드 컴포넌트에는 네트워크 요청과 백엔드 계약 상세를 직접 넣지 않는다.
- 백엔드 라우터는 요청 검증과 HTTP 입출력만 담당하고, 비즈니스 로직은 `BE/services/`에 둔다.
- LangGraph 그래프 구성, 상태, 노드 로직은 `BE/agents/` 아래에 둔다.
- DB 접근 로직은 `BE/crud/`, 데이터 모델은 `BE/models/`, API 입출력 스키마는 `BE/schemas/`에 둔다.

## Modification Rules
- 변경은 요청 범위에 직접 필요한 파일로 제한한다.
- 기존 구조와 책임 경계를 유지하면서 수정하고, 구조 이동이나 이름 변경은 필요할 때만 수행한다.
- 사용자가 만든 미확인 변경이나 현재 작업과 무관한 수정은 되돌리지 않는다.
- API 계약, 상태 필드, 공용 타입을 바꾸면 영향 받는 FE/BE 경로를 함께 점검한다.
- 레거시처럼 보이는 코드나 폴더도 실제 참조 여부를 확인하기 전에는 삭제하지 않는다.

## API And Streaming Rules
- 백엔드 API 라우터 prefix는 `/api/v1`를 기준으로 유지한다.
- 헬스 체크 엔드포인트는 `/health`를 사용한다.
- 스트리밍 엔드포인트는 `text/event-stream` 계약을 유지하고, 프론트엔드 소비 방식과 함께 변경한다.
- API 엔드포인트나 요청/응답 형태를 바꾸면 `BE/routers/`, `BE/services/`, `FE/lib/api.ts`를 함께 점검한다.

## Frontend Rules
- TypeScript 타입을 유지하고, props와 API 응답 형태를 명시한다.
- 상태 관리와 화면 로직은 기존 폴더 책임에 맞춰 유지한다.
- 스타일 변경은 기존 Next.js + Tailwind 기반 구조를 따른다.
- 컴포넌트에서 직접 임의의 백엔드 fetch를 추가하지 않는다.

## Backend Rules
- Python 함수에는 타입 힌트를 유지한다.
- 외부 I/O와 네트워크 호출은 비동기 흐름을 우선한다.
- 설정값과 비밀값은 환경변수로 관리하고 코드에 하드코딩하지 않는다.
- CORS와 프론트엔드 연동 계약을 깨는 변경은 이유 없이 하지 않는다.

## Review And Fix Workflow
- 변경 전에는 관련 코드와 실제 구조를 먼저 확인한다.
- 변경 후에는 먼저 review 관점으로 영향 범위, 계약 위반, 회귀 가능성을 점검한다.
- 발견한 문제는 같은 작업 흐름에서 바로 fix하고, 수정 후 다시 검증한다.
- review 결과 이상이 없을 때만 작업을 마무리한다.

## MCP Usage Rules
- MCP가 제공하는 저장소, 문서, 외부 시스템 정보가 있으면 임의 추정보다 우선한다.
- 읽기 전용 조회와 맥락 수집에 우선 사용하고, 쓰기 작업은 범위와 대상이 명확할 때만 수행한다.
- MCP 결과가 현재 워크트리와 충돌하면 로컬 파일 상태를 기준으로 다시 확인한다.
- 외부 서비스 변경 작업은 사용자가 명시적으로 요구한 경우에만 진행한다.

## Git And Commit Rules
- 커밋 메시지는 `{emoji} {Type}: {Description}` 형식을 사용한다.
- 커밋 메시지 설명은 영어로 작성한다.
- 문서 변경은 `📝 Docs: ...`, 기능 변경은 `✨ Feat: ...`, 버그 수정은 `🐛 Fix: ...` 형식을 우선 사용한다.
- push는 사용자가 명시적으로 요청한 경우에만 진행한다.
- 로컬 터미널에서 `git merge`로 브랜치를 직접 병합하지 않는다.
- 브랜치 병합이 필요하면 GitHub PR 또는 웹 UI 기반 머지를 우선하고, 다른 방식은 사용자가 명시적으로 요청한 경우에만 진행한다.

## Naming And Comments
- Python은 `snake_case`, TypeScript는 `camelCase`를 사용한다.
- 주석은 필요한 경우에만 짧게 작성하고, 기존 팀 관례에 맞춰 한국어를 우선한다.

## Safety
- API 키, 토큰, DB 접속 정보는 커밋하지 않는다.
- `main` 브랜치에 직접 반영하는 작업은 피한다.
- 파괴적 정리나 삭제는 실제 참조 여부와 영향 범위를 확인한 뒤에만 수행한다.
